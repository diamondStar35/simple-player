To trigger the auto-updater in your application, you must perfectly align the version number in your code, the `info.json` file on your `main` branch, and the asset files uploaded to your GitHub Release.

Here is the exact step-by-step process to publish a release so your app successfully auto-updates.

### Step 1: Update the Version in Your Code

Before compiling your app, update the version number inside `config/constants.py`.

```python
# Change this to your new version
APP_VERSION = "1.0.1" 

```

### Step 2: Build and Name Your Files Exactly

Your auto-updater relies on a strict naming template: `SimpleAudioPlayer-v{version}.{ext}`.
After compiling your app, rename your output files to exactly match the version you are releasing. For version `1.0.1`, you need:

1. **Portable Version:** `SimpleAudioPlayer-v1.0.1.zip`
* *Important:* Ensure the zip contains the main folder with your `.exe` and `.dll` files inside, and **no** `.sap_installed` marker file.


2. **Installer Version:** `SimpleAudioPlayer-v1.0.1.exe`
* *Important:* Your installer must create the `.sap_installed` marker file in the installation directory so the app knows to download `.exe` updates in the future.



### Step 3: Update `info.json` and Push to GitHub

Your app checks the `info.json` file on the `main` branch to see if an update exists.

1. Open `info.json` in the root of your project folder and update it:

```json
{
  "version": "1.0.1",
  "changes": [
    "Fixed YouTube playlist skipping bugs.",
    "Added Arabic translation.",
    "Improved audio silence removal."
  ]
}

```

2. Commit and push this change to your repository:

```bash
git add info.json config/constants.py
git commit -m "Bump version to 1.0.1 for release"
git push origin main

```

*Note: The moment you push this, older versions of your app will see the update. However, if the release files aren't uploaded yet (Step 4), the download will fail. Do Step 4 immediately.*

### Step 4: Publish the Release on GitHub

Now, create the release and upload the files you built in Step 2.

1. Go to your repository page on GitHub: `https://github.com/kamalyaser31/simple-player`
2. On the right-hand side, click on **Releases**.
3. Click the **Draft a new release** button.
4. **Choose a tag:** Type exactly **`v1.0.1`** (You *must* include the 'v' because your code formats the URL as `tag = f"v{version}"`). Click "Create new tag: v1.0.1 on publish".
5. **Release title:** Type something like `Update v1.0.1`.
6. **Description:** You can paste the same changes from your `info.json` here.
7. **Attach binaries:** Drag and drop `SimpleAudioPlayer-v1.0.1.zip` and `SimpleAudioPlayer-v1.0.1.exe` into the attachment box at the bottom.
8. Click the green **Publish release** button.

### How to Test It

To verify everything works, run an older version of your app (e.g., `1.0.0`) with **Check for app updates on startup** enabled in Preferences.

When the app launches, it will read `info.json`, detect version `1.0.1`, show you the popup with your changes, and download the correct file from the GitHub release you just published.
